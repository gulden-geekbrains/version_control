# Основные команды Git
### *Создание репозитория*
```sh
git init
Команда создает структуру Git репозитория в текущей папке
```

### *Создание контрольных версий файлов*
```sh
git add <filename>
Команда добавляет новый файл под версионный контроль и индексирует изменения

git commit -m <"Comment">
Команда фиксирует проиндексированные изменения и создает комментарий к ним, указанный пользователем в кавычках

git status
Команда выводит информацию о состоянии файлов в репозитории, наличии или отсутствии проиндексированных и закоммиченных изменений
 ```

 ### *Работа с каталогом изменений*
 ```sh
 git log
 Команда выводит перечень коммитов, сделанных в репозитории в обратном хронологическом порядке
 
 git log --oneline
 Краткий вид отображения перечня коммитов

 git log --graph
 Отображает создание и слияние веток графически, может использоваться совместно с git log --oneline

 git diff
 Команда выводит непроиндексированные изменения в отслеживаемых файлах

 git diff <commit> <commit>
 Команда выводит изменения между закоммиченными состояниями файлов

 git checkout <commit>
 Команда возвращает файлы в состояние, зафиксированное указанным коммитом

 git checkout master
 Команда возвращает файлы в состояние, соответствующее последнему зафиксированному изменению
 ```
 ### *Работа с ветвлением версий*

 ```sh
 git branch
 Вывод списка веток
 
 git branch <branch_name>
 Создание новой ветки

 git branch -d <branch_name>
 Удаление ветки
 
 git checkout <branch_name>
 Переход на существующую ветку

 git merge <branch_name>
 Слияние текущей ветки с указанной
 ```
### *Работа с удаленным репозиторием*
```sh
git clone <url>
Создание клона удаленного репозитория в локальной папке

git remote add origin <url>
Добавление удаленного репозитория с названием "origin"

git push -u origin <branch_name>
Команда направляет изменения указанной ветки в удаленный репозиторий

git pull
Команда стягивает изменения из удаленного репозитория в локальный и автоматически выполняет объединение веток
```
