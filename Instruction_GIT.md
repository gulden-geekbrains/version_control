# Инструкция по GIT

## Команды командной строки

для того, чтобы корректно воспользоваться GIT
необходимо в командной строке (или терминале)
указать папку в которой мы будем создавать репозиторий

* команда смены дертиктории CD

```sh
cd 
```

* Команда отображения текущей директории Win
```sh
dir
```

## Команды GIT
Находясь в директории, нужно инициировать работу GIT

```sh
git init
```

Командная строка покажет, что этой папке добавлен
репозиторий GIT

Теперь можно запрашивать статус GIT

```sh
git status
```

Если изменения были сделаны и были сохранены, то
строка покажет вам изменения, выделенные красным.
После этого необходимо добавить изменения в библиотеку GIT
```
git add Имя изменённого файла
```

*Используя TAB можно добавлять файлы после ввода первых символов*

Изменения будут сохранены в репозитории только после
того, как вы добавите комментарий к ним
```
git commit -m "Ваш комментарий"
```

## Другие полезные команды:
* Просмотреть все изменения
```
git log
```
* Просмотреть все изменения в упрощённом виде

```
git log -- oneline
```

* Перейти к тому или иному статусу

```
git checkout номер статуса или первые 4 цифры 16тизначного
номера
```

* Удалить файл или не сохранённый статус

```
git restore <file>
```

 ## Работа в ветках

 Для работы в ветках используются следующие команды:
 
 + создание веток
	```
	git branch (название ветки)
	```
	Команда создаёт ветку, для работы с черновиком по конкретному заданию.

+ Проверить наличие веток в git-файле

```
git branch
```

+ Перейти с ветки на ветку

```
git checkout (название ветки)
```

+ Для слияния веток необходимо сменить ветку на ту, куда будем заливать изменённые данные, затем использовать следующую команду
```
git merge (название ветки)
```

+ И наконец удаление той или иной ветки после слияния

```
git branch -d (название ветки)
-D для полного и безоговорочного удаления
<<<<<<< HEAD
<<<<<<< HEAD
```

## Работа с Github

Прежде всего толерантность и демократия. Именно поэтому первая команда при работе с github это переименовывание ветки master в ветку main. Отсылы на тёмное рабовлачельческое прошлое не допустимы!
```
git branch -M main
```
Важно при этом находиться в ветке master дабы не переименовать случайно другую ветку.
Только гендерно нейтральные и толерантные файлы должны находиться в святом теле github.

Теперь вы можете привязать ваш локальный репозиторий к github.
```
git remote add origin https://github.com/имя вашего репозитория
```

Теперь можем закачивать файлы на наш внешний репозиторий, используя команду origin.
```
git push -u origin main (или другое имя ветки) 
```

В случае, если вы хотите актуализировать вашу локальную версию, подгрузив в неё последние изменения с внешнего репозитория, вам необходимо воспользоваться следующей командой:
```
git pull
```

Можно управлять удалённым репозиторием также через локальный терминал
Например:
```
git push origin -- delete имя ветки, которую хотим удалить
```

Для того, чтобы скачать все изменения с сервера необходима следующая команда:
```
git pull --rebase
```

Команда для продолжения rebase
```
Git rebase --continue
```

После этого возможна команда ```git push ```

Для клонирования репозитория на компьютер
```
git clone "ссылка с сайта github"
```
