# Команды для работы с программой GIT

**1. Основные команды для работы с GIT**  
```sh
git init
инициализация нового репозитория

git add
добавление файлов к индексу для подготовки к коммиту

git commit -m "Сообщение комиита"
добавление коммита с указанным сообщением

git status
просмотр состояния файлов в рабочем каталоге и индексе

git lig (--oneline --graph)
просмотр истории коммитов

git diff [11ff11] [22gg22]
просмотр различий между версиями
```

**2. Создание веток, перемещения по commit**
```sh
git branch -b [name_branch]  
создает новую ветку и сразу на неё переключается

git branch -M [name_branch]
переименовывает текущую ветку

git rebase master  
перемещает(объединяет) текущую ветку в ветку master

git checkout master^  
перемещение на 1 commit назад (^^ на 2 и т.д.)

git checkout master~4 
перемещение на 4 commit назад
```

**3. Работа с удаленным репозиторием**
```sh
git clone [URL]
копирует на компьютер удаленный репозиторий

git remote add origin [URL]
устанавливает по умолчанию удаленный репозиторий

git remote [show (+origin)] [-v]
показывает привязанный удаленный репозиторий

git push -u origin main
отправляет (выталкивает) ветку с названием main на удаленный репозиторий

git push --set-upstream origin [name_branch]
отправляет (выталкивает) НОВУЮ ВЕТКУ, созданную в локальном репозитории

git push origin --delete [name_branch]
удаляет ветку из удаленного репозитория

git pull
запрос изменений с удаленного сервера

git pull --rebase
скачивает изменения с удаленного репозитория и производит слияние
```

**4. PULL REQUEST**  
```sh
1. Перейти на репозиторий с которым требуется взаимодействовать (на GITHIB например)
2. Нажать кнопку "Fork"
3. Создать лакальную копию (git clone)
4. Для изменений создать новую ветку
5. Отправить ветку с изменениями на удаленный репозиторий
6. В браузере нажать "Compare & pull requst"
7. Добавить сообщение и описание и нажать "create pull request"
```
