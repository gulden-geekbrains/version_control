# Инструкция по пройденным командам Git
## 1. Перед тем как использовать Git надо представится Gitu это можно сделать следующим образом:
```sh
git config --global user.name "Name"
```
```sh
git config --global user.email "Email"
```
### После чего представился можно проверить свои данные следующим способом:
```sh
git config user.name
```
```sh
git config user.email
```
## 2. Чтобы перейти в необходимый каталог (папку) нужно использовать следующую команду:
### Для Macos, Linux
```sh
cd ~/Desktop/name_file
```
### Для Windows
```sh
cd C:/user/name/Desktop/file_name
```
## 3. В Macos, Linus можно увидеть весь путь к файлу следующей командой:
```sh
pwd
```
## 4. Листинг текущей директории (эти команды покажут файл или папку с которой вы работаете):
### Для windows:
```sh
dir
```
### Для Macos, Linux:
```sh
ls
```
## 5. С помощью следующей команды можно создать новый репозиторий (скрытая папка для работы) ***Git***.
```sh
git init
```
## 6. Показывает состояние рабочего каталога и проиндексированного снимка состояния ( показывает изменения в файле с которым вы работаете)
```sh
git status
```
## 7. Переносит изменения из рабочего каталога в раздел проиндексированных файлов
```sh
git add
```
## 8. Получает проиндексированный снимок состояния и выполняет его коммит в историю проекта (комментирует проиндексированы файл для облегчения работы с проектом и ориентацией в нем)
```sh
git commit -m "text"
```
## 9. Чтобы удалить ненужный файл воспользуемся следующими командами:
### Для Windows
```sh
del <name_file>
```
### Для Linux, Macos
```sh
rm <name_file>
```
## 10. Позволят изучить предыдущие версии проекта.  Эта команда имеет несколько параметров оформления, с помощью которых можно отобразить снимки состояния после выполнения коммитов.
```sh
git log
```
## 11. Также с помощью следующей команды можно укоротить список.
```sh
git log --oneline
``` 
## 12. С командой *git checkout* можно не только получать старые коммиты и прежние версии файлов, но и осуществлять навигацию по существующим веткам. В сочетании с базовыми командами *Git* она позволяет сосредоточиться на определенном направлении разработки.
```sh 
git checkout 
```
## 13. С помощью этой команды можно вернутся к последнему состоянию
```sh
git checkout master
```  
## 14. Сравнение — это функция, анализирующая два входных набора данных и отображающая различия между ними.
```sh
git diff
```
## 15. Эта команда выступает универсальным инструментом администрирования веток. С ее помощью можно создавать изолированные среды разработки в одном репозитории.
```sh
git branch
```
## 16. Эффективный способ интеграции изменений из разошедшихся веток. После разветвления истории проекта командой ***git branch*** можно использовать команду ***git merge***, чтобы объединить отдельные ветки.
```sh
git merge
```

## 17. С помощью этой команды можно переходить (перемешаться) по ветке _**branch**_ используя следующую команду:
```sh
git checkout name_branch_list
```
## 18. Если необходимо удалить из ветки *__branch__* один из черновиков используем следующую команду:
```sh
git branch -d name_list
```

## 19. Параметр ***--graph*** используется для построения диаграммы из символов ASCII, отражающей структуру веток в истории коммитов. Обычно его используют в сочетании с командами ***--oneline***.
```sh
git log --oneline --graph
```
# Работа с удаленными репозиториями
## Для того чтобы работать с удаленным репозиторием необходимо скопировать особенную ссылку:
HTTPS
```sh
https://github.com/gulden-geekbrains/version_control.git
```
## Чтобы сделать правильно клон репозитория нужно использовать следующий код:
```sh
git clone https://github.com/gulden-geekbrains/version_control.git
```        
## После копирования репозитория появится папка с рабочим проектом чтобы начать с ней работать надо перейти в нее с командой :
```sh
cd name_file
```
## После чего мы можем начать выполнять работу в этом репозитории.

## После чего мы перешли в папку репозитория можем выполнить команду *__git status__* который нам покажет что в скопированном репозитории ветка будет называется *__main__*, а не __*master*__ как в локальном репозитории (в проекте который мы создаем с нуля на персональном компьютере). Также мы видим что у нас появилась в описании ветка обновлена ищо одна надпись перед *__main__*. *__origin__* - это по умолчанию синоним удаленного репозитория.



# Регистрация, и первый репозитории  

## Но чтобы начать полноценно работать с *__GitHub.com__* надо зарегистрировался. После регистрации мы можем создать на сайте свой первый репозиторий. Для этого на своей странице находим кнопочку с названием *__New__* нажимаем на нее и переходим на страницу где будем создавать свой репозитории. На странице мы увидим пустую ячейку для заполнения с названием *__Repository name__* мы туда записываем имя проекта. Смотрим чтобы было выбрано пункт *__Public__* иначе не хто не увидит и не сможет в дальнейшем вам помогать в проекте , или повторить его, спускаемся ниже и нажимаем кнопку *__Creating repository...__*.    

## Чтобы начать работать с репозиторием который мы создали нужно выполнить инструкцию которую нам выдает сам сайт __*GitHub*__.

## Чтобы связать локальный и удаленный репозитории нужно выполнить следующую команду:
```sh
git remote add origin ссылка на репозитории 
```
## Чтобы отправить свою версию (измененную, обновленную) репозиторию необходимо использовать следующую команду:
```sh
git push -u origin main
```
## После выполнения этой команды вас попросят авторизоваться (связать git с __*GitHub*__) это в том случае если вы будете использовать ссылку *__https://__* но если вы сделаете связь с помощью ключа *__ssh__* то у вас просто ваш репозиторий отправится без проблем в __*GitHub*__.

## Чтобы вытянуть репозитории из __*GitHub*__ надо использовать команду:
```sh
git pull 
```
## С помощью команды _**git push**_ можно отправлять не только главные ветки __*main*__ но и другие которых мы сделали но нужно переключится не них с помощью команды *__git checkout name_branch__*

## После чего мы отправили все ветки на удаленный репозиторий мы можем сделать слияние и удалить ненужную ветку (черновик). Удаление веток можно сделать как в браузере так и с помощью команды при отправке ветки мы можем дополнить команду следующим образом:
```sh
git push origin --delete name_branch
```
## При конфликте когда не можно отправить свое изменения ( git push ) мы должны сделать
```sh
git pull --rebase
```
## Ключ __*--rebase*__ он скачает изменения и сразу же сделает слияния с нашим локальным репозиторием. И после скачивание и слияние и решения конфликтов и сделали индексацию и коммит. Мы можем отправить наш измененный репозиторий.

## Когда работают над большим проектом много людей, и чтобы было меньше конфликтов надо по чаше набирать ***git pull***.

# Как подключится к разработке и иметь собственную копию проекта.

## Для того чтобы подключится к чужому проекту и сделать собственную копию необходимо перейти на страницу проекта и там нажать на кнопочку *__fork__*. И после чего мы сделали копию на собственный *__GitHub__* мы уже можем от туда сделать клонирование на персональный ПК. Когда репозиторий будет на нашем персональным ПК мы можем туда скопировать готовый файл проекта который мы хотим предложить разработчику проекта но эго надо добавлять в новую ветку и туда вносить изменения после чего как мы уже практиковали раньше отправим командой *__git push__* на собственный *__GitHub__* и там нам предложит сам сайт сделать коммит и отправить наше предложения разработчику проекта.
