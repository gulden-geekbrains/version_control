# Инструкция по языку MarkDown
## Что такое Git?
**Git** - это специальная программа, которая позволяет отслеживать любые изменения в файлах, хранит их версии и оперативно возрващаться в любое сохораненное состояние. Большинство других систем контроля версий хранят информацию в виде списка изменений в файлах. 

## Подготовка Git
Создаем папку, в которой будет хранится наша информация по Git.

Для создания репозитория необходимо выполнить команду *git init* в папке с репозиторием и у нас появится скрытая папка .git

## Создание коммитов
### Git add
Для добавления изменений в коммит используется команда *git add*. Чтобы использовать команду *git add* нужно в терминале написать *git add <имя файла>*

### Просмотр состояния репозитория 
Для того, чтобы посмотреть состояние репозитория используется команда *git status*. Для этого необходимо в папке с репозиторием написать *git status*, и увидим были ли изменения в файлах, или их не было.

### Создание коммитов
Для того, чтобы создать коммит(сохраненеие) необходимо выполнить команду *git commit*. Выполняется она так:
*git commi -m "комментарий"*

### Журнал изменений
Перед переключением версии файла в Git, используем команду *git log*, чтобы увидеть количество сохранений и комментарии к этим сохранениям.

### Переключение между версиями
Для работы указываем не только интересующий коммит(сохраненеие) *git checkout + уникальный идентификационный номер*. Для ввода УИН не обязательно указывать полный номер, стоит указать лишь первые 7 чисел.
> Уникальный идентификационный номер состоит из 40 цифр (хоть там есть буквы, но они обознаются как цифры, так как используется шестнадцатиричная система счисления) 

Благодаря этой команде *git checkout* можно вернуться в версию, где работали при помощи полной команде *git checkout master*

### Разница между текущим и сохраненным файлом
При вводе команды *git log*, показываются зафиксированные изменения, которые были сохранены. 

Также есть команда *git log --oneline*, грубо говоря, это сокращенный результат команды *git log*, в которой показываются первые 7 чисел уникального идентификационного номера сохранения и его комментарии.

### Отличия модификаций (без коммита)
Команда *git diff* показывает отличия от модификации (без коммита) и уже сохраненного (с коммитом)
Также указываются удаленные(-) и добавленные(+) строки.



## Стилизация текста
Чтобы выделить курсивом текст, мы используем звездочку (*) или нижнее подчеркивание (_).
Например, *звездочка* и _подчеркивание_

А для того, чтобы выделить текст полужирным, мы используем две звёздочки (**) или два нижних подчеркивания (__). Например, **звездочки** и __подчеркивания__

Также используется стиль выделения текст, как зачеркивание. Оно обозначается двумя тильдами (~~). Например, ~~тильда~~

Эти способы существуют для того, чтобы у пользователя могли быть варианты выделения текста одновременно зачеркиванием, курсивом и полужирным. Например, *мы используем ~~этот~~ текст для __инструкции__*
Таким образом мы выделили текст курсивом, сделали зачеркивания слова, а главное слово полужирным.

## Цитирование в MarkDown
Чтобы параграф отобразился как цитата, нужно поставить перед ним закрывающую угловую скобку (>)
Например:
> Первый уровень цитирования

Внутри одного блока цитаты можно поместить несколько параграфов и использовать любые элементы оформления. 
Например:

>### Здесь у нас заголовок
> Соотвественно первый параграф
>
> Второй параграф
>
>> Вложенная цитата
>>> Цитата третьего уровня
>
> Продолжаем основную цитату
 
## Списки
В синтексисе MarkDown есть несколько видов списков. Для их оформления перед каждым пунктом необходимо поставить подходящий тег и отделить его от текста пробелом.

### Ненумерованный список
Для создания ненумерованного списка нужно поставить перед каждым пунктом звездочку (*), дефис (-) или плюс (+).
Например:
* Лист 1
* Лист 2
- Лист 3
- Лист 4
+ Лист 5
+ Лист 6

### Нумерованный список
Для создания нумерованного списка перед пунктами нужно поставить число с точкой. При этом нумерация в разметке ленива, и неважно, какие именно числа поставят. MarkDown пронумерует список автоматически.
Например:

>1. Лист 1
>2. Лист 2
>3. Лист 3

Или же

>1. Лист 1
>8. Лист 2
>13. Лист 3

Список можно начинать и не с единицы. Для нумерации важно только число, которое стоит перед первым пунктом.
Например:

> 32. Лист 1
> 33. Лист 2
> 34. Лист 3

Если списки идут по очереди, то нужно отступить от предыдущего списка две строки.  Но если пропустить только одну, то MarkDown воспримет список как один. 
Например:
>1. Лист 1
>2. Лист 2
>3. Лист 3
>
>1. Лист 4
>2. Лист 5
>3. Лист 6

## WEB ссылки
Самы легкий способ добавить ссылки, это зако=лючить их в угловые скобки <>.
Например, <https://gb.ru/>

Чтобы оформить ссылкой часть текста, используются такой синтаксис:

[Образовательная платформа](https://gb.ru/ "GeekBrains") с подсказкой

[Образовательная платформа](https://gb.ru/ ) без подсказки

## Изображения в тексте

Изображения в MarkDown оформляются по принципу, схожему с принципом оформления ссылок, только перед квадратными скобками нужно проставить восклицательный знак. Также можно сделать всплывающую подсказку
Например: 
![GeekBrains](https://gb.ru/channels/base/images/company/logo-geekbrains.svg "Логотип GeekBrains")

## Работа с удалённым репозиторием

Начнем с понятий **Git & GitHub**

>*Git* - одна из систем контроля версий, способная поддерживать и организовывать версионности определенной задачи.

>*GitHub* - самый популрный сервис Git компании Майкрософт для организации работы удаленных репозиторием, который способствуют архивации огромного количества различных кодов.

### Команды 

*git clone* - команда, позволяющая скопировать удаленный репозиторий на свой ПК, а также она является составной. Данная команда не только загружает все изменения, но и пытается слить все ветки на локальном компьютере и в удаленном репозитории.

Например, _наименование компьютера + **gitclone** + ссылка на адрес удаленного репозитория_
 
*git pull* - команда, которая скачивает все текущие изменения из удалённого репозитория на локальный репозиторий, автоматически объединяя.
