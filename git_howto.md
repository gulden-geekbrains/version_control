# Инструкция для работы с MarkDown

## Выделение текста

Чтобы выделить текст курсивом необходимо обрамить его звёздочками (*) или знаком нижнего подёркивания (_). Например, *вот так* или _вот так_.

Чтобы выделить текст полужирным необходимо обрамить его двойными звёздочкамит (**) или двойным знаком нижнего подчёркивания (__). Например **вот так** или __вот так__.

Альтернативные способы выделения текста жирным или курсивом нужны для того, чтобы мы могли совмещать оба этих способа. Например, _текст может быть выделен курсивом и при жтом быть **полужирным**_.

## Списки
Чтобы выделить ненумерованный список, используйте (*)

Чтобы добавить ненумерованные списки необходимо пункты выделить звёздочкой (*). Например вот так:

* Элемент 1
* Элемент 2
* Элемент 3
* Элемент 4

Чтобы добавить нумерованные списки, необходимо пункты просто пронумеровать. Например, вот так:
1. Первый пункт
2. Второй пункт

## Работа с изображениями

Чтобы вставить изображение в текст, достаточно написать следующее:

![Привет, это тефтелька!](Teftelka.jpg)

# Инструкции по работе  с GIT и удалёнными репозиториями
## Что такое **GIT** ?
Git - это одна из систем контроля версий, имеющая как локальные, так и удалённые репозитории. Является самой популярной реализацией систем контроля версий.

## Подготовка репозитория.
Для создания репозитория необходимо выполнить команду _**git init**_ в папке с репозиториями и у Вас создастся репозиторий (появится скрытая папка git)

```sh
git init
```

## Добавление файла в просматриваемые *.GIT*
Для того чтобы __*.git*__ начал *"следить"* за нашим файлом, нужна следующая команда:
```sh
git add file_name
```
где file_name - это имя нашего файла.

## Создание коммитов.
Чтобы сохранить (_зафиксировать_) последнее изменение файла мы выполняем команду:

```sh
git commit -m "Наш комментарий"
```

В дальнейшем мы будем перемещаться между версиями нашего кода (_коммитами_). Это необходимо, чтобы видеть все изменения, которые происходили с файлом.

Комментарий лучше писать на английском языке.

Крайне важно оставлять *информативный* комментарий, чтобы другие коллеги могли понять, какое именно изменение я зафиксировала.

## Как увидеть все изменения файла (через коммиты - **commits**)?
Для этого есть команда:
```sh
git log
```
*Данная команда* помогает нам увидеть все сохранённые версии изменения нашего файла. 

Здесь я могу посмотреть информацию по имени создателя commit, дате создания коммита, и комментарий к коммиту.

## Как увидеть коммиты в более коротком виде?
Для этого есть следующая команда:
```sh
git log --oneline
```

## Как перемещаться между коммитами и для чего это нужно?
Перемещение между коммитами необходимо, чтобы увидеть все изменения нашего файла. Эта функция также облегчает коллективную работу. В коммите можно увидеть имя работника, и изменения, которые он внёс.

Команда следующая:
```sh
git checkout первые 4 цифры коммита
```

## Как посмотреть разницу между определёнными коммитами (*commits*)?
Если разница между двумя коммитами (commits) есть, то это значит, что с момента первого до следующего коммита произошли изменения. Их то и выдаёт следующая команда:

```sh
git giff
```
Если после этой команды ничего не происходит, то изменения не произошли. 

## Как вывести список веток в репозитории?
```sh
git branch
```
Данная команда мне нужна, чтобы посмотреть мои ветки и увидеть на какой ветке сижу я __*сейчас*__.

## Как создать новую ветку с именем new_branch_name?
```sh
git branch new_branch_name
```
где _**new_branch_name**_ - имя новой ветки, которую создаю.

## Как удалить ненужную нам ветку?
```sh
git branch -d branch_to_delete
```
где **branch_to_delete** - это имя нашей ветки, которую хотим удалить.

## Как перейти на другую ветку?
```sh
git checkout branch_name
```
где **branch_name** - это имя ветки, на которую мы хотим перейти.

Когда я перехожу на другую ветку, я возвращаюсь к тому состоянию, в котором осталась эта ветка.

## Как вывести список коммитов в виде красивого графа/дерева?
```sh
git log --graph
```

## Как вывести список коммитов в виде графа/дерева в укороченном виде?
```sh
git log --oneline --graph
```

## Как "слить" ветку branch_name с текущей веткой?
```sh
git merge branch_name
```

В этом случае всё содержимое ветки branch_name перейдёт в ветку master.

# Работа с удалёнными репозиториями. GitHub.
## Как направить свой локальный репозиторий в GitHub и репозиторий из GitHub на локальный компьютер. 
1. Создать аккаунт на GitHub
2. Создать локальный репозиторий.
3. "Подружить" (синхронизировать) свой локальный и удалённый репозиторий. GitHub при создании нового репозитория подскажет как это сделать.
4. Отправить (push) свой локальный в удалённый на GitHub, при этом возможно нужно будет авторизоваться на удалённом репозитории. НО, если я уже синхронизировала GitHub и локальный компьютер, то далее это уже не нужно делать.
Команда:
```sh
git push 
```
5. Провести изменения уже со своего аккаунта на GitHub.
6. "Выкачать" (pull) актуальное состояние из удалённого репозитория на локальный компьютер
```sh
git pull
```

## Как предложить свои изменения в существующий проект
1. Делаю **FORK** интересующего нас репозитория, после чего данный репозиторий появится у меня на моем аккаунте GitHub.
2. Копирую данный репозиторий с аккаунта на GitHub в локальный компьютер с помощью команды **git clone** скопированной ссылки на своем аккаунте в GitHub
```sh
git clone 
```
3. Создаю ветку с своими предлагаемыми изменениями
```sh
git branch имя_ветки
```
4. Производим все изменения _**только**_ в этой ветке.
5. Далее отправляем эти изменения на свой аккаунт с помощью команды:
```sh
git push
```
6. Далее в окне на GitHub появляется возможность отправить **pull request**.
