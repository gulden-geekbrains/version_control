# Инструкция для работы с Git

## Что такое Git?
Git - это одна из реализаций  распределённых систем  контроля версий, имеющая как локальные так и удаленные  репозитории. Является самой популярной реализацием систем контроля версий в мире.

## Подготовка репозитория
Для создания репозитория нужно ввести команду *git init* (появится скрытая папка git.)

## Создание коммитов

### Добавить файл в репозиторий
Нужно ввести *git add "имя файла"* (можно использовать tab)

Если ввести *git add.* будут добавлены все файлы в этой области

### Просмотр состояния репозитория
Для того чтобы посмотреть состояние репозитория используется команда *git status*.

## Создание коммитов
Для создания коммита испольуется команда *git commit -m “message”* 

## История коммитов
Команда *git log* - вывод на экран истории всех коммитов с их хеш-кодами

## Переход по кммитам
*git checkout* – переход от одного коммита к другому

*git checkout master* – вернуться к актуальному состоянию и продолжить работу

*git diff* – увидеть разницу между текущим файлом и закоммиченным файлом

## Синтаксис языка Markdown

Заголовок – выделение заголовков. Количество символов “#” задаёт уровень заголовка
(поддерживается 6 уровней).

 = или - – подчёркиванием этими символами (не менее 3 подряд) выделяют заголовки первого(“=”) и второго (“-”) уровней.

 ** Полужирное начертание** или __ Полужирное начертание__

 *Курсивное начертание* или _Курсивное начертание_
 ***Полужирное курсивное начертание***

 ~~Зачёркнутый текст~~ (~~)
 * Строка – ненумерованные списки, символ “*” в начале строки

 1, 2, 3 … – нумерованные списки

 Справочник по Markdown от Microsoft:
https://docs.microsoft.com/ru-ru/contribute/markdown-reference

# Ветки

 git branch – посмотреть список веток в репозитории

 git branch <название ветки> – создать новую ветку

 git checkout <название ветки> – переход к другой ветке

 git branch -d <название ветки> – удалить ветку

## Работа с Github.com

git clone - копировать репозиторий

git push - отправильно локальный репозиторий в удаленный
git pull - скачать актуальное состояние из удаленного репозитория
