# Инструкция для работы с Git и удаленными репозиториями

## Что такое Git?
Git - это одна из реализаций распределенных систем контроля версий, имеющая как и локальные , так и удаленные репозитории. Является самой популярной реализацией систем контроля в мире.
## Подготовка репозитория
Для создания репозитория необходимо выполнить команду *git init* в папке с репозиторием и у Вас создасться репозиторий (появится скрытая папка .git)

## Создание коммитов
### Git add
Для добавления изменений в коммит используется команда
*git add*. Чтобы использовать команду *git add* напишите *git add <имя файла>*

## Просомтр состояние репозитория
### Git status
Для того, чтобы посмотреть состояние репозитория используется команда *git status*. Для этого необходимо в папке с репозиторием написать *git status*, и Вы увидите были ли измения в файлах, или их не было.

## Создание коммитов
### Git commit
Для того, чтобы создать коммит(сохранение) необходимо выполнить команду *Git commit*. Выполняется она так:
необходимо написать *git commit -m message*, где *message* - это сообщение которое оставляем к коммиту.

## Просмотр текущей ветки
### git branch
Для того, чтобы просмотреть текущуюю ветку необходимо выполнить команду *git branch*. Для этого необходимо в папке с репозиторием написать *git branch*, и вы увидите в какой ветке сейчас находитесь (помечено звездочкой).

## Создание новой ветки
### git branch name
Для того, чтобы создать новую ветку необходимо выполнить команду *git branch* Для этого необходимо написать *git branch name*, где *name* - это имя которое мы хотим присвоить ветке.

## Смена ветки
### git checkout
Для того чтобы сменить ветку на другую необходимо выполнить команду *git checkout*. Для этого необходимо написать *git checkout name*, где *name* - это название ветки, присвоенное ранее.

## Объединение веток
### git merge name
Для того чтобы объеденить ветки необходимо выполнить команду *git merge*. Для этого необходимо написать *git merge name*, где *name* - это название ветки, присвоенное ранее. Таким образом в ветку в которой мы сейчас находимся перемещается то что было в указанной ветке.

## Удаление веток
### git branch -d name
Для того чтобы удалить ненужную ветку необходимо выполнить команду *git branch -d*. Для этого необходимо написать *git branch -d name*, где *name* - это название ветки, присвоенное ранее.

# Работа с удаленными репозиториями
## Клонирование удаленных репозиториев
### git clone https
Для того чтобы клонировать репозиторий необходимо выполнить команду *git clone*. Для этого необходимо написать *git clone https*, где *https* - это URL адрес скопированный с сайта github.com

## Создание удаленного репозитория
### New
Для того чтобы созданить новый удаленный репозиторий, после регистрации на сайте github.com, необходимо нажать на кнопку "new". После этого задать имя репозитория не используя пробелы.

## Переименование главной ветки
### git branch -M main
Github рекомендует в качестве основого названия ветки использовать "main". По этому рекомендуется при работае с удаленными репозиториями переименовывать основную ветку. Для этого необходимо написать *git branch -M main*, где *main* - это название ветки, которую хотим присовить.

## Настройка удаленного репозитория по умолчанию
### git remote add origin https
Для того чтобы задать удаленный репозиторий по умолчанию для локального репозитория в котором находимся в текущий момент, необходимо выполнить команду *git remote add origin https*, где *https* - это URL адрес созданного нами ранее удаленного репозитория.

## Передача информации с локального репозитория на удаленный
### git push -u origin main
Для того чтобы отправить информацию с локального репозитория на удаленный, необходимо выполнить команду *git push -u origin main*, где *origin main* - вытолкнуть изменения в удаленный сервер "origin" на ветку "main". Команда *git push* выталкивает на сервер ту ветку, в которой сейчас находимся.

## Проверка привязки удаленного репозитория
### git remote -v
Для того чтобы проверить привязку удаленного репозитория, необходимо выполнить команду *git remote -v*. После этого будет указано два репозитория:
1. "fetch" - тот с которого принимаем изменения
2. "push"  - тот на который отправляем изменения
Для более подробной информации можно использовать команду *git remote show origin*

## Изменение удаленного репозитория через браузер
Для того чтобы отредактировать файл, необходимо зайти в него через браузер, сохранить изменения и оставить комментарий к изменениям

## Прием информации с удаленного репозитория
### git pull
Для того чтобы получить последние изменения с удаленного репозитория, необходимо выполнить команду *git pull*. После этого, если не было конфликтов, изменения синхронизируются. Если будет конфликт - необходимо выбрать ту версию которую необходимо сохранить. Для того чтобы одновременно скачать изменения с сервера и записать их необходимо воспользоваться командой *git pull --rebase* , после разрешения конфликта необходимо воспользоваться командой *git rebase --continue*.
