# How to work with git


main commands

```
1. git init - инициализация локального репозитория
2. git status - получить информацию от git о его текущем состоянии
3. git add - добавить файл или файлы к следующему коммиту
4. git commit -m “message” – создание коммита(git commit -a -m "commit" - эта команда позволяет сохранить и закоммитить все документы сразу)
5. git log - вывод на экран истории всех коммитов с их хеш-кодами
6. git checkout - переход от одного коммита к другому(git checkout -b "название ветки" - Создаем ветку и сразу же переходим в нее, выполняя сразу 2 действия в одной комнаде)
7. git checkout master - вернуться к актуальному состоянию и продолжить работу
8. git diff - увидеть разницу между текущим файлом и закоммиченным файлом
9. git branch -d "название ветки" - удалить метку ветки
```

```sh
1. Добавление новых файлов в репозиторий. 
Избранные файлы: 
    git add file1 file2 file3
2. Получение изменений из главного репозитория: 
    git pull
3. Отправка изменений в главный репозиторий (с авторазрешением конфликтов): 
    git pull (проверить на наличие новых изменений в репозитории и, если они есть, выкачать их и объединить с локальными изменениями)
    git push (отправить изменения в репозиторий)
```
