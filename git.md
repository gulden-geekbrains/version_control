# Подсказка по git

##Создание репозитория
```sh
git init
```

## Создание версий
### Добавить файл
```sh
git add <имя файла>
```
### Для комментирования
```sh
git commit -m "комментарий"
```
### Сброс изменений
```sh
git restore <имя файла>
```
## Просмотр статуса
```sh
git status
```
## Просмотр лога изменений
```sh
git log
```
Показывает кто и когда внёс изменения, а также идентификатор версии

Если добавить --oneline после log то список станет компактнее
Если добавить --graph после log то список станет информативнее

## Переход между версиями или ветками
```sh
git checkout <идентификатор версии или название ветки>
```
Если вместо идентификатора поставить master, вернёт последнюю версию. Достаточно первых 4 символов идентификатора
## Просмотр изменений
```sh
git diff <идентификатор 1> <идентификатор 2>
```
## Просмотр веток и добавление веток
```sh
git branch просмотр веток
git branch <новая ветка> добавление ветки
git branch -d <имя ветки> удаление метки
```
## Сведение веток
```sh
git merge <название ветки>
```
## Работа с удалёнными репозиториями
```sh
git remote add origin <ссылка на репозиторий>
git push -u <удалённый репозиторий> <ветка> отправить ветку на сервер
git pull получить данные с сервера
если добавить --rebase, то при конфликте сохранятся локальные изменения
```
